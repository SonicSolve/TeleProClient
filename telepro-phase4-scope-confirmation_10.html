In Jesse's words: "I want to see where I have issues, address the issues, and stop the bleeding." Phase 4 is the financial and operational visibility layer — one place where every manager can see, per project, where the money stands, where the risk is, and what needs attention right now.

The Story We Are Telling

WHAT THIS NEEDS TO ANSWER

Phase 4 is not a technology project. It is a visibility problem. Right now, the information exists — it lives in Structure, in Smartsheet, in Jorge’s weekly template. But it does not come together in one place where a manager can look and immediately know what needs attention. These are the questions Phase 4 needs to answer, in the order they matter.

01
Where do I have issues?
At any given time, across 300+ active projects, which ones need attention right now? Not tomorrow — right now. The system should surface these automatically so managers are not digging to find them.
02
Am I over budget — or over contract?
Over budget is a warning. Over contract is a crisis. Jesse was clear: “My contract is $100K, my budget was $50K, and my cost today is $102K — big problem.” The system needs to show this difference, instantly, per project.
03
Is anyone sandbagging?
When a team member reports strong numbers but actual spending and hours tell a different story, that is a red flag. Jesse wants to catch the pattern early — not at 95% when it is too late to course-correct.
04
Who is working on what?
People naturally gravitate toward the jobs that are going well and avoid the ones in trouble. The manager view needs to make it impossible to ignore a struggling project — it should be the first thing that surfaces, not the last.
05
Are we going to finish strong?
“The last 10% to get somebody to finish strong is the hardest part.” The 90% milestone flag exists because Jesse knows from experience that is where jobs fall apart. The system needs to put all hands on notice before it becomes a problem.
06
Are we stopping the bleeding?
This is how Jesse described the entire Phase 4 goal: “I want to see where I have issues, address the issues, and stop the bleeding.” Every piece of this build — every dashboard, every alert, every flag — serves that one purpose.
Part 01

WHAT HAS BEEN BUILT

The following work has been completed as the foundation for Phase 4. This is the infrastructure everything else is built on top of.

Data Pipeline from Structure — Established
Weekly Thursday imports from Structure into Smartsheet via Data Shuttle across all company entities: Tele-Pro (TLP), SafeGuard Fire (SGF), SafeGuard Security (SGS), and SafeGuard Keys (SGK).
Approved Dataset Aligned — Template V6 Confirmed
The approved cost template (V6) has been confirmed as the dataset standard. The Project Health Tracker columns have been mapped to match this template exactly. See Part 02 for the full column alignment.
Project Health Tracker — Built and Aligned
The central tracking sheet captures all approved data points per project per scope category. Each column maps directly to a field in the approved template and serves a defined purpose in the Phase 4 reporting structure.
Centralized Data Table — Established
A single database holds all Structure cost records by company, category, phase, and transaction date. Resolves row-limit issues and allows data to be filtered and split into any report or dashboard without losing records.
Per-Scope Portfolio Dashboards — Built
Individual overview dashboards built for every system type: Data, A/V, CCTV, Intercom, Fire Alarm, Intrusion, Access Control, CATV, Gates, Fire Sprinkler, IT, HVAC Controls, Emergency Phone, MPWR, and more.
Per-Project Dashboard — Framework in Place
Every provisioned project has its own dashboard shell with data connections in place. Remaining work is completing widget mapping from the confirmed dataset. Search-by-job-ID experience is being built into this view.
Cost Breakdown Dashboard — Versions 1.0 and 2.0
Both versions are in the Phase 4 Assets workspace. Version 2.0 is the current active version, being refined as category alignment is finalized.
Category Mapping Matrix — Underway
A mapping matrix bridges the naming differences between Structure cost categories and Smartsheet budget structure. Validation against active projects is in progress. This directly unblocks full dashboard accuracy.
